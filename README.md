# 哈夫曼文件压缩器

基于哈夫曼编码的无损文件压缩工具，支持任意二进制文件（文本、图片、可执行文件等）的压缩与解压。

---

## 📈 特点

- 完整实现压缩+解压全流程
- 模块化设计，六大模块清晰分离
- 支持任意二进制文件
- 可扩展性强

---

## 功能

| 模块 | 功能 | 说明 |
|------|------|------|
| 统计频率 | 读取文件，统计 256 种字节（0-255）的出现频率 | 为建树提供数据基础 |
| 构建哈夫曼树 | 使用优先队列（小根堆）构建最优前缀编码树 | 贪心策略，高频字符靠近根 |
| 生成编码 | DFS 遍历树，为每个字符生成变长二进制编码 | 左走加"0"，右走加"1" |
| 压缩 | 将编码后的比特流打包成字节，写入压缩文件 | 满8位合并，不足补零 |
| 解压 | 读取频率表重建哈夫曼树，逐位解码还原原始文件 | 按原始大小精确截断 |

## 压缩文件格式

```
┌─────────────────┬─────────────────┬─────────────────┐
│  原始文件大小 4B  │   频率表 1024B   │   压缩后的数据   │
│   (1个int)       │  (256个int)      │  (变长比特流)   │
└─────────────────┴─────────────────┴─────────────────┘
```

| 字段 | 大小 | 用途 |
|------|------|------|
| 原始文件大小 | 4 字节 | 解压时精确控制输出长度，忽略尾部补零 |
| 频率表 | 1024 字节 | 256 个 `int`，解压时重建与压缩时完全相同的哈夫曼树 |
| 压缩数据 | 变长 | 哈夫曼编码拼接后按字节打包 |

## 六步通关

```
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│ 读文件   │ → │ 建哈夫曼 │ → │ DFS得   │ → │ 查表变   │ → │ 位运算  │ → │ 解压   │
│ 统计频率 │   │ 树      │   │ 编码表  │   │ 0101串  │   │ 压成字节 │   │ 还原   │
└─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘
   freq[]        树结构       a→0         "010100"      0x51 81        原文件
                              b→10
                              c→11
```

## 核心算法

- **建树**：贪心策略，每次合并频率最小的两个节点，时间复杂度 O(n log n)
- **编码**：前缀码性质，无任何编码是其他编码的前缀，保证无歧义解码
- **打包**：比特流缓冲，满 8 位通过位运算 `<<` 和 `|=` 打包成字节
- **解压**：逐位读取，0 往左走，1 往右走，到叶子节点输出字符

---

## 注意事项

| 问题 | 说明 |
|------|------|
| 内存泄漏 | 哈夫曼树节点使用 `new` 分配，程序结束时未释放 |
| 空文件 | 未处理空文件情况，可能导致崩溃 |
| 字节序 | 频率表按本机字节序存储，跨平台解压可能不兼容 |
| 小文件 | 固定 1028 字节头部开销，可能比原文件还大 |
